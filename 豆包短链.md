🚀 快速部署（推荐方案一）
1. 创建两个仓库
仓库 1：前端页面（url-shortener）
用于存放 HTML 页面和跳转逻辑
仓库 2：数据库仓库（url-shortener-db）
仅用于存储 Issues，不需要任何代码
2. 上传核心文件
我已经为您提供了两个核心文件：
404.html - 核心跳转逻辑（最重要）
index.html - 首页展示页面
3. 修改配置
编辑 404.html 中的配置项：
javascript
运行
const CONFIG = {
    // 替换为你的数据库仓库API地址
    GITHUB_API_URL: 'https://api.github.com/repos/yourusername/url-shortener-db/issues/',
    TIMEOUT: 5000,
    CACHE_DURATION: 60
};
4. 启用 GitHub Pages
进入前端仓库的 Settings 页面
滚动到 GitHub Pages 部分
选择 Source 为 main branch
点击 Save 保存
📚 使用方法
添加短链接
访问数据库仓库的 Issues 页面
创建新 Issue，标题填写目标 URL（必须包含 http (s)://）
例如：https://www.example.com/long-url
访问短链接
格式：https://yourdomain.com/[issue编号]
示例：如果 Issue 编号是 123，访问 https://short.yourdomain.com/123
🌐 自定义域名配置
域名解析：添加 CNAME 记录指向 yourusername.github.io
创建 CNAME 文件：在前端仓库根目录创建 CNAME 文件
GitHub 配置：在仓库设置中配置自定义域名
